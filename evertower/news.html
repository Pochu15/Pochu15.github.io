Hello world! update

15/6/2020

Testing, testing... it works! Welcome to the first Evertower Lite Development post! In these posts, I'll explain the changes I made to the game as well as the features I plan to develop.

What I've done

So far, Evertower Lite has:

  • A permanent upgrades system, which allows you to gain permanent buffs like extra health or unlock new spells.
  • An inventory system, with support for both passive and active items.
  • A spell system. Each character has a deck where spells are held. At the beggining of each battle, that deck is shuffled. Players can only use spells from their hands (and will draw spells at the beggining of each turn), and these spells are shuffled back in the deck after finishing the battle.
    • Spells, items and permanent upgrades work with a list of effects. Each spell has a list of effects that are activated when the spell is casted.
  • A random dungeon generation system, built so that it's easy to add dungeon elements to the generation pool. Each dungeon currently has:
    • Chests, which will give the player loot.
    • Enemies, which will trigger battle mode.
    • Doors, which will take you from one point in the dungeon to another.
    • Crafting tables, which will allow you to craft unlocked recipes.
  • A fully functioning Battle system.
    • Although no items in the game currently implement this, each battling character has an event system. As for now, there's only six battle-events: On battle start, On turn start, On turn end, After taking damage, Before casting a spell and after casting a spell. When any of this events is triggered, it'll activate all of the effects associated to it. This opens the door to very interesting features like poison damage, which would cause damage at the end of each turn, or getting healed up after dealing damage. The possibilities this enables are endless.
  • Data persistancy. Although there's still room for improvement, player data is stored. Concretely, a permanent upgrades data file will be created, and will be loaded at the beggining of the game. If the player quits while doing a dungeon, data will also be stored. There's no save button yet, but data is stored when you end a battle, when you open a chest or when you craft something. Dying will erase your dungeon data.

The contents of the game

There's currently not much content in the game, as there's only five different spells, five different items, two enemy types and 10 different permanent upgrades. Here's a peek at some artwork!

What I want to do next

  • Add more content to the game. New spells, new items, new permanent upgrades, new enemies...
  • Finish (or rather, start) the Shopping system, allowing the player to spend gold to get items from a random pool
  • Add boss fights, and different zones with different structure and different enemies.
  • Add multiple levels (A stair to the next level), and make the enemies stronger based on the current level.
  • Add Legendary Items to the game. Curious? Here's a design!
  • Add new elements to the basement. I'd like to have something to allow the player to customize its starting deck, and maybe an anvil to allow the player to upgrade its spells.
